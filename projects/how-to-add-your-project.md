# Como Adicionar Seu Projeto

Todos os projetos do Hack4Freedom São Paulo 2026 são documentados neste repositório para que possam perdurar além do hackathon.

Cada equipe é responsável por adicionar um resumo do seu projeto à pasta `/projects`.

Isso ajuda a:
- Mostrar seu trabalho publicamente
- Tornar seu projeto mais fácil de ser descoberto
- Apoiar o desenvolvimento contínuo, grants e oportunidades de trabalho

---

## O Que Você Precisa Fazer

Cada equipe deve adicionar **um arquivo markdown** descrevendo seu projeto.

Você não precisa de permissão — basta seguir os passos abaixo.

---

## Instruções Passo a Passo

### 1. Faça um Fork do Repositório

- Acesse o repositório Hack4Freedom São Paulo 2026
- Clique em **Fork** (canto superior direito)

---

### 2. Crie Seu Arquivo de Projeto

Dentro da pasta `/projects`:

- Crie um novo arquivo nomeado com o nome do seu projeto
- Exemplo:
  ```
  lightningo.md
  bitsave.md
  ```

---

### 3. Copie o Modelo de Projeto

Copie o conteúdo de `project-template.md` para o seu novo arquivo.

Preencha todas as seções relevantes:
- Visão geral
- Problema
- Solução
- Stack de tecnologia
- Membros da equipe
- Links para o repositório no GitHub e demo
- Status atual e próximos passos

Você não precisa ser perfeito — clareza é mais importante que polimento.

Exemplo:
```markdown
# LightningO

**Hack4Freedom São Paulo 2026**

---

## Visão Geral

LightningO é uma plataforma que conecta patrocinadores globais a builders locais através de bounties, gigs e grants usando a Lightning Network e o Nostr.

O objetivo é criar oportunidades transparentes e sem fronteiras para desenvolvedores no Brasil e no Sul Global.

---

## Problema

Muitos desenvolvedores talentosos no Sul Global têm dificuldade de acessar oportunidades de financiamento globais devido a:
- Barreiras de pagamento
- Dependência de plataformas centralizadas
- Falta de visibilidade internacional
```

---

### 4. Faça o Commit das Suas Mudanças

- Faça commit do seu novo arquivo com uma mensagem clara

---

### 5. Abra um Pull Request

- Abra um pull request de volta para o repositório principal
- Coloque o título:
  ```
  Add <Nome do Projeto> project
  ```

Pronto 🎉

---

## Diretrizes

- Os projetos devem ser **open-source**
- Inclua o link para o repositório real no GitHub onde o código está
- Use linguagem clara e simples
- Isto é documentação, não um pitch deck

---

## Precisa de Ajuda?

Se você travar em algo:
- Abra uma Issue neste repositório
- Ou peça ajuda no canal da comunidade Hack4Freedom

Aprender a contribuir com documentação faz parte da experiência open-source.
